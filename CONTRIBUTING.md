# Contributing

## Purpose

This repository uses a documentation governance model. Changes should improve clarity, consistency, accessibility, or maintainability.

## Contribution process

1. Create a branch.
2. Make your changes.
3. Run Vale.
4. Open a pull request.
5. Explain what changed and why.

## Pull request checklist

Before requesting review, confirm that your changes:

- Follow the style guide
- Use approved terminology
- Include examples where helpful
- Use accessible headings and link text
- Pass Vale checks
- Do not introduce conflicting rules

## Governance model

Style guide changes should be reviewed for:

- Consistency with existing rules
- Impact on writers and reviewers
- Impact on user comprehension
- Maintenance cost
- Tooling impact
