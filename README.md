# Documentation Style Guide

A documentation style guide for technical content teams. This project defines content standards for voice and tone, headings, procedures, UI text, API documentation, accessibility, plain language, terminology, and automated style checks with Vale.

## Purpose

This guide helps documentation teams create clear, consistent, accessible, and maintainable content. It is designed for technical writers, product teams, support teams, engineers, and documentation reviewers.

## What this guide covers

- Voice and tone
- Heading rules
- Procedure rules
- UI text rules
- API documentation rules
- Accessibility and plain-language rules
- Approved terminology
- Vale linting rules for automated style checks

## Who this is for

This guide is intended for teams that create:

- Help center articles
- Developer documentation
- API references
- Product release notes
- Onboarding guides
- Knowledge base articles
- User interface copy
