# Documentation Style Guide

## Core principles

Documentation should be:

- Clear
- Accurate
- Consistent
- Accessible
- Task-focused
- Easy to scan
- Easy to maintain

Write for users who are trying to complete a task, solve a problem, or understand a system. Do not make users work harder than necessary.

## Voice

Use a voice that is:

- Helpful
- Direct
- Calm
- Professional
- Human

Avoid sounding robotic, overly casual, or overly promotional.

### Preferred

> Select **Save** to apply your changes.

### Avoid

> Simply smash that save button and you’re good to go!

## Tone

Adjust tone based on the user’s situation.

| Situation | Tone |
|---|---|
| Setup task | Clear and encouraging |
| Error state | Calm and specific |
| Security warning | Direct and serious |
| Release note | Brief and factual |
| API reference | Precise and consistent |

## Headings

Headings should help users scan the page.

Use sentence-style capitalization.
